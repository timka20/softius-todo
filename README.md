# TODO List ООО "Софтиус"

Серверная часть построена на фреймворке Express.js с СУБД SQLite (через драйвер better-sqlite3). 
Реализованы REST API эндпоинты:
```
  • POST /auth/login — авторизация с проверкой хеша пароля через bcrypt
  • POST /auth/logout — выход из системы с удалением сессионной cookie
  • GET /auth/me — получение текущего авторизованного пользователя
  • PUT /auth/profile — редактирование профиля (ФИО, должность, телефон)
  • PUT /auth/password — смена пароля с завершением всех сессий
  • GET /assignable-users — список пользователей, на которых можно назначить задачу
  • POST /users — создание пользователя (только администратор)
  • GET /users — список всех пользователей (только администратор)
  • GET /workers — список работников
  • DELETE /users/:id — удаление пользователя (только администратор)
  • GET /tasks — получение задач с учётом роли пользователя
  • GET /tasks/notifications — ближайшие дедлайны и назначенные задачи
  • GET /tasks/:id — получение задачи по идентификатору
  • POST /tasks — создание задачи
  • PUT /tasks/:id — редактирование задачи
  • PATCH /tasks/:id/progress — обновление прогресса задачи
  • DELETE /tasks/:id — удаление задачи (только администратор)
  • GET /tasks/:id/history — история изменений конкретной задачи
  • GET /history — общая история изменений с учётом прав доступа
  • GET /export/csv — экспорт задач и истории в CSV 
```

## Инстуркция по запуску:

1. Установите зависимости
Перейдите в папку проекта и выполните команду:
```
npm install:all
```
2. После завершения установки зависимостей, запустите проект
```
npm run dev 
``` 

## Стек и описание работы
Разработка велась с использованием современного стека технологий: клиентская часть построена на React 18 с TypeScript и сборщиком Vite, интерфейс оформлен с помощью Tailwind CSS и библиотеки иконок Lucide React; серверная часть реализована на платформе Node.js с использованием фреймворка Express и TypeScript, а данные хранятся в базе данных SQLite (better-sqlite3). Приложение оформлено как прогрессивное веб-приложение (PWA): добавлены манифест, иконки и Service Worker для возможности установки на мобильные устройства. Все этапы — от проектирования архитектуры и базы данных до тестирования и оптимизации — были выполнены в соответствии с техническим заданием.
